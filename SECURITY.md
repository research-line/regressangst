# Security Policy

## Deutsch

Auch wenn dieses Repository primär Forschungsdokumente enthält, können sicherheitsrelevante
Probleme auftreten, z. B. durch ungewollt veröffentlichte sensible Daten, problematische
Artefakte oder missverständliche Build-/Tooling-Anweisungen.

### Bitte öffne kein öffentliches Issue für

- versehentlich eingecheckte sensible Dateien oder Zugangsdaten
- personenbezogene Daten in Dokumenten oder Assets
- Schwachstellen in Hilfsskripten oder Build-Schritten mit Schadpotenzial

### Meldeweg

1. Nutze nach Möglichkeit GitHub Private Vulnerability Reporting
2. Falls das nicht verfügbar oder unpassend ist, melde dich an `hallo@um-bruch.org`
3. Beschreibe kurz Fundstelle, Risiko und mögliche Reproduktion

---

## English

Although this repository is primarily a research publication repo, security-relevant
issues can still occur, for example through accidentally published sensitive files,
problematic artifacts, or unsafe tooling instructions.

### Please do not open a public issue for

- accidentally committed secrets or sensitive files
- personal data contained in documents or assets
- vulnerabilities in helper scripts or build steps with abuse potential

### Reporting path

1. Prefer GitHub Private Vulnerability Reporting when available
2. If that is unavailable or not appropriate, contact `hallo@um-bruch.org`
3. Include the location, the risk, and short reproduction details
